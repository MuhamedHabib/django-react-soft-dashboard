## Overview

This repository is a **two-tier full-stack starter**: a Django REST API backend (`django-api`) decoupled from a React single-page admin dashboard (`react-ui`). The frontend is the **Soft UI Dashboard React** theme, and the two pieces communicate over a simple JSON Web Token authentication flow.

> **Credit / template basis.** This project is **not original work** — it is based on the open-source [Django React Soft Dashboard](https://appseed.us/product/django-react-soft-dashboard) starter generated by **[AppSeed](https://appseed.us/)**. The React UI is **[Soft UI Dashboard React](https://github.com/app-generator/react-soft-ui-dashboard)** by **[Creative Tim](https://www.creative-tim.com/)** & AppSeed, and the backend is the **[Django API Server](https://github.com/app-generator/api-server-django)**. All credit for the original design and code belongs to Creative Tim and AppSeed. This repo is a clone/customization starting point — see their pages for the canonical product, demo, and support.

## Features

- **Two-tier architecture** — React frontend decoupled (logically and physically) from the Django API backend.
- **JWT authentication flow** — register, login, logout, and active-session check, implemented end-to-end on both backend and frontend.
- **Django REST API** — built with Django 4 + Django REST Framework, SQLite by default (engine configurable via environment), with `django-cors-headers` enabled for the React origin.
- **Soft UI admin dashboard** — Material UI (MUI v5) based React UI with a custom `Sui*` component library (Box, Button, Input, Badge, Alert, Avatar, Progress, Pagination, Typography) and ready-made example components (Cards, Charts, Tables, Navbars, Sidenav, Timeline, etc.).
- **Prebuilt layouts** — dashboard, profile, billing, tables, authentication, virtual reality, and RTL pages.
- **Containerized** — `Dockerfile` + `docker-compose.yml` provided for both the API and the UI.

## Tech Stack

**Backend (`django-api`)**
- Django `4.0.1`
- Django REST Framework `3.13.1`
- PyJWT `2.3.0` (JWT auth)
- django-cors-headers `3.11.0`
- SQLite by default (DB engine read from `DB_ENGINE` environment variable)

**Frontend (`react-ui`)**
- React `17` + React Router DOM `5`
- Material UI (MUI) `5` with Emotion styling
- Axios for API calls
- Chart.js + react-chartjs-2 for charts
- Build tooling: `react-scripts` (Create React App), ESLint + Prettier

## Getting Started

> Requirements: Python 3 + pip for the backend, Node.js + npm (or Yarn) for the frontend. Docker is optional.

### Backend — Django API

```bash
cd django-api

# (optional) create & activate a virtual environment
python -m venv env
# Windows:  .\env\Scripts\activate
# Unix:     source env/bin/activate

pip install -r requirements.txt
python manage.py migrate
python manage.py runserver 5000
```

The API will be available at `http://localhost:5000`. Configuration such as `DEBUG`, `DB_ENGINE`, and `DJANGO_ALLOWED_HOSTS` is read from environment variables — see `django-api/env.example` for the supported keys (copy it to your own `.env` and fill in your own values).

A Docker path is also available:

```bash
cd django-api
docker-compose up --build
```

### Frontend — React UI

```bash
cd react-ui
npm install      # or: yarn
npm start        # or: yarn start
```

The UI starts on `http://localhost:3000` and expects the backend on `http://localhost:5000`. The backend base URL is set in `react-ui/src/config/constant.js` (`API_SERVER`) — change it there if your API runs elsewhere.

### Authentication endpoints

The DRF router exposes the auth surface under `/api/users/`:

| Method | Endpoint | Purpose |
| --- | --- | --- |
| `POST` | `/api/users/register` | Create a new account |
| `POST` | `/api/users/login` | Obtain a JWT |
| `POST` | `/api/users/logout` | Invalidate the session (sends the token in the `Authorization` header) |
| `POST` | `/api/users/checkSession` | Validate an active session |
| `PUT`  | `/api/users/edit` | Edit the current user |

## Notes

- **No secrets in this repo.** Do not commit a real `SECRET_KEY`, database credentials, or JWT secrets. Use environment variables (`env.example` documents the keys) and keep your `.env` out of version control.
- **Default database is SQLite** and `DEBUG` defaults to on — both are fine for local development but must be changed before any production deployment.
- **CORS** is preconfigured to allow `http://localhost:3000` / `http://127.0.0.1:3000`; update `CORS_ALLOWED_ORIGINS` in `core/settings.py` for other origins.
